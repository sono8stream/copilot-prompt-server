# copilot-instructions.md

## 実装方針（必須）

1. 新機能・挙動変更の実装前に、まず `chat-ui-spec.md` を確認する。  
2. 仕様に曖昧さ・矛盾・不足がある場合は、実装を進める前に仕様を更新して合意を取る。  
3. 実装・修正は、仕様の受け入れ条件を満たすことをゴールにする。  
4. 仕様外の回避策（例: 自動フォールバックを標準化）を入れる場合は、事前に仕様へ明記する。  

## このリポジトリにおけるチャット実装上の優先順位

1. 自動フォールバック遷移は行わず、単一の明示された方式で動作させる。  
2. 対話ウィンドウのUX（会話継続性、状態可視化）を優先する。  
3. 応答のリアルタイム逐次表示を優先する。  
4. 障害時の挙動は、黙って切り替えるのではなくUI上で明示する。
