# チャットUI仕様（合意事項）

この仕様は、ユーザー要望に基づく必須要件を整理したものです。  
以降の実装は本仕様を先に確認し、逸脱しないこと。

## 1. 目的

- サーバー上の選択フォルダを作業ディレクトリにして Copilot CLI と対話する。
- Web UI 上で、**対話ウィンドウとして自然に会話が進む体験**を提供する。
- 応答生成中の状況が、**リアルタイムに逐次表示**されること。

## 2. 必須要件

1. **自動フォールバック遷移を通常動作にしないこと**
   - 「常駐対話方式」が標準動作。
   - 無応答時に自動的に別方式へ切り替える設計は、原則として採用しない。
   - 例外的にフォールバックを用いる場合は、明示的な仕様合意が必要。

2. **対話ウィンドウをUIとして表示すること**
   - ユーザー発話とAI発話が時系列で確認できるチャット領域を提供する。
   - 入力中/送信中/応答中の状態がUIで判別できること。

3. **途中経過をリアルタイム表示すること**
   - AI応答は、生成完了後にまとめて表示ではなく、生成中のテキストを段階的に表示する。
   - 「回答中」状態の可視化（インジケータ、経過時間等）を行う。

## 3. 禁止・非期待挙動

- 無応答時にユーザーへ通知せず内部方式を切り替える。
- 生成中に何も表示されず、完了時にのみ一括表示される。
- チャットとしての会話継続性がUI上で確認できない状態。

## 4. 受け入れ条件

- ユーザーがメッセージ送信後、短時間で「回答中」表示に遷移する。
- 応答本文が逐次追記される（リアルタイム感がある）。
- 完了時に同じ吹き出しが確定表示される（不自然な消去/再描画がない）。
- 方式変更や障害時は、ユーザーが理解できる形で明示される。
